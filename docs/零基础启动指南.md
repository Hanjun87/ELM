## 准备工作

### 需要安装的软件

#### 1. Python（后端需要）
- **Windows**: 访问 https://www.python.org/downloads/，下载最新版 Python（建议 3.11+）
  - 安装时**勾选** "Add Python to PATH"（重要！）
- **macOS**: 
  ```bash
  # 打开终端（Terminal），安装 Homebrew，然后安装 Python
  /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  brew install python@3.11
  ```

#### 2. uv（Python 包管理工具）
- 打开命令行（Windows: 按 `Win+R`，输入 `cmd`；macOS: 打开 Terminal）
- 执行：
  ```bash
  # Windows (在 PowerShell 中执行)
  powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
  
  # macOS/Linux
  curl -LsSf https://astral.sh/uv/install.sh | sh
  ```

#### 3. Node.js（前端和小程序需要）
- 访问 https://nodejs.org/
- 下载并安装 **LTS 版本**（长期支持版）
- 安装完成后，打开命令行执行以下命令验证：
  ```bash
  node --version
  npm --version
  ```

#### 4. 微信开发者工具（只有小程序需要）
- 访问 https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
- 下载对应系统的**稳定版**
- 安装后打开，用微信扫码登录

## 启动后端服务

### 步骤 1：打开项目目录

**Windows:**
1. 在文件资源管理器中找到 `ELM` 文件夹
2. 在地址栏输入 `cmd` 并回车，会打开命令行窗口

**macOS:**
1. 打开 Terminal（终端）
2. 输入 `cd ` 然后把 `ELM` 文件夹拖到终端窗口，回车

### 步骤 2：进入后端目录并安装依赖

```bash
cd src/elm
uv sync
```

**这一步会做什么？** 自动安装后端需要的所有 Python 库，大约需要 1-3 分钟。

### 步骤 3：初始化数据库

```bash
uv run python manage.py migrate
uv run python manage.py init_data
```

**这一步会做什么？** 
- `migrate`: 创建数据库表结构
- `init_data`: 添加测试账号和示例商家数据

### 步骤 4：启动后端

```bash
uv run python manage.py runserver
```

**成功标志：** 你会看到类似这样的输出：
```
Django version 5.x.x, using settings 'config.settings'
Starting development server at http://127.0.0.1:8000/
Quit the server with CTRL-BREAK.
```

**✅ 保持这个窗口打开！** 后端需要一直运行。

**如何测试：** 浏览器访问 http://localhost:8000/api/v1/merchants/，应该能看到商家列表的 JSON 数据。

## 启动 Web 管理后台

### 步骤 1：新开一个命令行窗口

**重要：** 后端窗口保持运行，再开一个新窗口！

- **Windows**: 按 `Win+R`，输入 `cmd`，回车
- **macOS**: 按 `Cmd+N` 新建终端窗口

### 步骤 2：进入 Manager 目录

```bash
# 先回到项目根目录（根据你的实际路径调整）
cd /path/to/ELM

# 进入 Manager 前端目录
cd fronted/Manager
```

### 步骤 3：安装依赖

```bash
npm install
```

**这一步会做什么？** 下载前端需要的所有库，第一次需要 3-5 分钟。如果很慢，可以换国内镜像：
```bash
npm install --registry=https://registry.npmmirror.com
```

### 步骤 4：启动 Manager

```bash
npm run dev
```

**成功标志：** 你会看到：
```
  VITE v6.x.x  ready in xxx ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: use --host to expose
```

### 步骤 5：打开浏览器访问

1. 浏览器访问：http://localhost:3000
2. 你会看到管理后台登录页面
3. 输入测试账号（见下方[测试账号](#测试账号)）
4. 登录后可以看到：
   - **数据看板**：订单统计、商家数量、骑手状态
   - **用户管理**：查看/封禁用户
   - **商家审核**、**财务结算**、**横幅管理**（这三个是 Mock 数据）

**📸 截图位置：** 登录后的数据看板页面、用户管理页面

## 启动微信小程序

我们有三个小程序：**客户端**、**商家端**、**骑手端**。这里以**客户端**为例，其他两个操作一样。

### 步骤 1：编译小程序代码

**新开一个命令行窗口**，进入客户端目录：

```bash
# 回到项目根目录
cd /path/to/ELM

# 进入客户端小程序目录
cd apps/customer

# 安装依赖
npm install

# 编译小程序（会监听代码变化，自动重新编译）
npm run dev:weapp
```

**成功标志：** 你会看到：
```
✔ Build success in xxx ms
```

编译产物在 `apps/customer/dist/` 目录，这就是要导入微信开发者工具的代码。

**✅ 保持这个窗口打开！** 修改代码后会自动重新编译。

### 步骤 2：打开微信开发者工具

1. 启动**微信开发者工具**
2. 点击左侧 **+（新建项目）**
3. 填写项目信息：
   - **项目名称**：随便填，例如 "ELM客户端"
   - **目录**：点击「选择」，找到 `ELM/apps/customer` 文件夹（**注意不是 dist 文件夹**）
   - **AppID**：
     - 如果没有小程序账号，点击「使用测试号」
     - 如果有自己的 AppID，就填自己的
4. 点击「确定」

### 步骤 3：配置开发环境

**重要配置：** 因为我们要连本地后端（localhost），需要关闭域名校验：

1. 点击右上角「详情」
2. 点击「本地设置」标签
3. **勾选**「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」

**不勾选这个会导致小程序无法访问后端！**

### 步骤 4：查看小程序效果

现在你应该能看到客户端小程序的界面了！

**主要页面：**
- **首页**：商家列表，可以搜索、筛选
- **订单**：我的订单列表（待付款、进行中、已完成）
- **购物车**：加入的商品
- **我的**：个人中心，可以管理地址、查看优惠券

**📸 截图位置：**
1. 登录页（点击「客户账号」快速登录）
2. 首页商家列表
3. 点击某个商家进入商家详情页
4. 购物车页面
5. 订单列表页面

### 启动商家端小程序（可选）

重复上述步骤，只需把目录改成 `apps/merchant`：

```bash
cd apps/merchant
npm install
npm run dev:weapp
```

微信开发者工具导入 `ELM/apps/merchant` 目录。

**主要页面：**
- **订单**：接单、拒单、出餐、完成
- **商品**：商品管理，上下架、编辑
- **数据**：营业数据（Mock）
- **我的**：店铺设置

**📸 截图位置：** 订单管理页面、商品管理页面

### 启动骑手端小程序（可选）

```bash
cd apps/rider
npm install
npm run dev:weapp
```

微信开发者工具导入 `ELM/apps/rider` 目录。

**主要页面：**
- **待接单**：可抢单列表
- **我的配送**：当前配送任务（取餐、送达）
- **历史**：已完成订单
- **我的**：骑手信息

**📸 截图位置：** 待接单页面、配送页面

## 常见问题

### Q1: 后端启动报错 "No module named 'xxx'"
**原因：** 依赖没装好  
**解决：** 回到 `src/elm` 目录，重新执行 `uv sync`

### Q2: 前端/小程序启动报错 "Cannot find module xxx"
**原因：** node_modules 没装好  
**解决：** 删除 `node_modules` 文件夹和 `package-lock.json`，重新 `npm install`

### Q3: 小程序白屏/报 request 错误
**原因：** 
1. 后端没启动
2. 没勾选「不校验合法域名」

**解决：**
1. 确认后端窗口在运行，浏览器访问 http://localhost:8000/api/v1/merchants/ 能看到数据
2. 微信开发者工具 → 详情 → 本地设置 → 勾选域名校验选项

### Q4: 小程序编译失败 "ERESOLVE unable to resolve dependency tree"
**原因：** npm 版本太新，依赖冲突  
**解决：** 用 `--legacy-peer-deps` 强制安装
```bash
npm install --legacy-peer-deps
```

### Q5: 微信开发者工具导入项目后提示 "dist 目录不存在"
**原因：** 还没编译  
**解决：** 先在命令行执行 `npm run dev:weapp` 或 `npm run build:weapp`，等编译完成后再导入

### Q6: 想同时打开客户端、商家端、骑手端三个小程序
**限制：** 如果三个小程序用的是同一个 AppID（测试号），微信开发者工具**无法同时打开**  
**解决：**
- 方法 1：注册三个不同的小程序 AppID
- 方法 2：分别截图后关闭再打开下一个

### Q7: 真机预览时小程序连不上后端
**原因：** 手机访问不了电脑的 localhost  
**解决：**
1. 确保手机和电脑在**同一个 WiFi**
2. 查看电脑的局域网 IP：
   - Windows: `ipconfig`，找到 `IPv4 地址`
   - macOS: `ifconfig | grep "inet "`，找到类似 `192.168.x.x` 的地址
3. 修改小程序的 API 地址：
   - 打开 `apps/customer/src/api/config.ts`（或 merchant/rider）
   - 把 `API_BASE_URL` 改为：`http://你的IP:8000/api/v1`
   - 例如：`http://192.168.1.100:8000/api/v1`
4. 重新编译：`npm run build:weapp`
