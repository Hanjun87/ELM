**版本**: 1.0  
**日期**: 2026-07-08  
**状态**: 已实现

---

## 一、项目背景

ELM 是一个面向本地生活服务的外卖配送平台，支持餐厅在线接单、骑手即时配送、客户手机下单，以及平台方对商家、用户和订单进行集中管理。系统采用前后端分离架构，提供四个独立的角色端界面。

---

## 二、系统角色

| 角色 | 英文标识 | 说明 |
|------|---------|------|
| 客户 | customer | 通过 Customer 端进行商品浏览、下单、支付、评价 |
| 商家 | merchant | 通过 Merchant 端管理店铺、商品、订单、查看评价 |
| 骑手 | rider | 通过 Rider 端接单、配送、更新状态 |
| 管理员 | admin | 通过 Manager 端管理用户、商家、查看平台数据 |

## 四、非功能需求

### 4.1 安全

| 要求 | 实现方式 |
|------|---------|
| 接口认证 | JWT Bearer Token |
| 权限控制 | RBAC，`IsAuthenticated` / `IsAdmin` 装饰器 |
| 数据隔离 | ORM 过滤确保商家/骑手只能看到自己的数据 |
| 角色白名单 | 注册接口仅允许 customer/merchant/rider |
| 并发安全 | 抢单使用 `select_for_update()` + `transaction.atomic()` |
| 生产配置 | `settings_prod.py` 强制 DEBUG=False、HTTPS、SECRET_KEY 非空 |

### 4.2 数据完整性

| 要求 | 实现方式 |
|------|---------|
| 下单库存扣减 | 行锁 + 原子操作 |
| 取消/拒单库存回滚 | `_restore_stock()` 同步回滚 stock 和 sales_count |
| 起送金额校验 | 下单时服务端检查 `total_amount >= merchant.min_order` |
| 价格校验 | 价格由服务端从商品表读取，不信任前端传入 |
| 默认地址原子性 | `transaction.atomic()` 先清除再设置 |

### 4.3 性能（开发阶段目标）

| 指标 | 目标 |
|------|------|
| API 响应时间 | < 500ms（P95） |
| 并发用户 | 开发阶段不做压测，生产阶段通过 Gunicorn/Daphne 多进程扩展 |
