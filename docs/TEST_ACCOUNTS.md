# 测试账号 & Mock 数据说明

> 本文档列出所有预置测试账号和 Mock 数据概况，方便快速登录体验各角色功能。

---

## 一、初始化数据命令

```bash
# 基础数据（4个测试账号 + 1个商家 + 基础分类）
python manage.py init_data

# 扩充数据（20个额外商家、10个额外客户、10个额外骑手、订单/评价/优惠券）
python manage.py add_more_data
```

> 两个命令均已在 `handle()` 开头把 stdout 重配为 UTF-8，避免 Windows GBK 控制台在输出 `✓`/emoji 时抛 `UnicodeEncodeError` 中断种子流程。

**Docker 环境**：首次启动时 `docker/entrypoint.sh` 自动执行 `init_data`。需要更多数据时手动运行：
```bash
docker-compose exec backend python manage.py add_more_data
```

## 四、Mock 数据概况

执行 `init_data` + `add_more_data` 后的实际数据规模：

| 数据类型 | 数量 | 说明 |
|---------|------|------|
| 👤 客户 | **11** | 1个初始 + 10个扩充 |
| 🏪 商家 | **21** | 1个初始 + 20个扩充，每个商家 3-6 个商品 |
| 📦 商品 | **98** | 每个商家随机 3-6 个商品，覆盖主食/小食/饮品/甜点等分类 |
| 🛵 骑手 | **11** | 1个初始 + 10个扩充 |
| 📋 订单 | **19** | 覆盖全部 8 个状态（pending/paid/accepted/preparing/ready/picked/delivered/finished） |
| ⭐ 评价 | **5** | 针对已完成订单（delivered/finished）的真实评价 |
| 🎫 优惠券 | **6** | 满减券、新人券等，部分已领取 |

> 订单/评价数量随机（前 8 个客户各随机生成 2-5 单，评价仅挂在已完成订单上），重跑 seed 后具体数字会浮动，商家/商品/骑手/客户数量固定。

### 订单状态分布

Mock 订单覆盖全部业务流程状态：
- **pending**: 待支付（刚创建）
- **paid**: 已支付，等待商家接单
- **accepted**: 商家已接单
- **preparing**: 备餐中
- **ready**: 出餐完成，等待骑手取餐
- **picked**: 骑手已取餐，配送中
- **delivered**: 已送达
- **finished**: 订单已完成

## 五、快速体验场景

### 场景 1：客户下单流程

1. 登录客户端（13800000001 / customer）
2. 浏览商家列表，选择任意商家（如"肯德基"）
3. 加购商品，确认订单
4. 模拟支付
5. 查看订单状态（等待商家接单）

### 场景 2：商家接单流程

1. 登录商家端（13800000002 / merchant）
2. 查看待处理订单（status=paid）
3. 接单 → 确认出餐 → 出餐完成
4. 订单进入 ready 状态，等待骑手

### 场景 3：骑手配送流程

1. 登录骑手端（13800000003 / rider）
2. 查看可接单列表（status=ready）
3. 抢单
4. 确认取餐 → 确认送达
5. 订单进入 delivered 状态

### 场景 4：管理员后台

1. 登录管理端（13800000004 / manager）
2. 查看平台数据看板（GMV、订单数、商家数、用户数）
3. 管理用户（查看列表、封禁/解封）
4. 管理商家（查看列表、强制下架）

## 六、数据重置

如需清空并重新初始化数据：

```bash
# 手动部署
rm src/elm/db.sqlite3
python manage.py migrate
python manage.py init_data
python manage.py add_more_data

# Docker 环境
docker-compose down -v  # ⚠️ 会删除数据库数据卷
docker-compose up -d    # 重新启动，自动执行 init_data
docker-compose exec backend python manage.py add_more_data
```

## 七、注意事项

1. **密码统一规则**：
   - 客户账号密码：`customer`
   - 商家账号密码：`merchant`
   - 骑手账号密码：`rider`
   - 管理员账号密码：`manager`

2. **商家状态**：所有 Mock 商家默认为 `open`（营业中），可通过商家端或管理端切换为 `closed`。

3. **订单时间戳**：Mock 订单的创建时间、支付时间等均为相对当前时间的偏移量，符合真实业务逻辑。

4. **优惠券**：部分客户账号已自动领取优惠券，可在客户端"我的优惠券"查看。

5. **评价**：已完成订单（delivered/finished）的部分有真实评价，商家可在商家端查看并回复。
